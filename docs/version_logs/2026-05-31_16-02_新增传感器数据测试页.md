# 2026-05-31 16:02 新增传感器数据测试页

## 本次修改目标

按照师兄给出的 `DKY_APITest.py` 功能说明，在 Android 端新增一个独立的“传感器数据测试页”，用于在手机上单独调试传感器接口，不影响现有机器人控制主界面。

## 修改了哪些文件

- `app/src/main/java/com/example/mavlinktest/SensorDataTestActivity.kt`
- `app/src/main/java/com/example/mavlinktest/SensorLineChartView.kt`
- `app/src/main/res/layout/activity_sensor_data_test.xml`
- `app/src/main/AndroidManifest.xml`
- `app/src/main/res/values/strings.xml`
- `docs/version_logs/2026-05-31_16-02_新增传感器数据测试页.md`

## 每个文件主要改了什么

- `SensorDataTestActivity.kt`：新增传感器接口测试页面逻辑，支持认证、拉取六路原始数据、解析 CSV、拉取损伤结果、表头排序和损伤等级颜色区分。
- `SensorLineChartView.kt`：新增自定义六通道折线图，支持手势缩放、左右拖动、底部滑块平移和通道显示/隐藏。
- `activity_sensor_data_test.xml`：新增传感器测试页面 UI，包括检测编号输入、认证、获取数据、六路曲线页、损伤列表页、通道开关和结果表格。
- `AndroidManifest.xml`：新增独立启动入口 `SensorDataTestActivity`，手机桌面会出现“传感器测试”入口。
- `strings.xml`：新增 `sensor_app_name`，用于传感器测试入口名称。

## 是否新增功能

是。新增独立传感器数据测试功能：

- `POST /auth/token` 获取接口 token。
- `GET /api/download/{serial}` 下载原始 CSV 数据。
- `POST /api/damageResult` 获取损伤结果列表。
- 六通道原始数据折线显示。
- 损伤结果按表头排序。
- 损伤等级按颜色区分。

## 是否修复 bug

否。本次主要是新增独立功能页。

## 是否还有遗留问题

- 当前页面使用自定义 Canvas 折线图，已经覆盖缩放、平移和通道隐藏需求；如果后续需要更复杂的数据游标、导出图片或专业坐标轴，可以再换成 MPAndroidChart。
- 夹紧/升降台/机器人控制逻辑没有和该页面合并，符合当前“单独在手机上跑通”的阶段目标。
- 本地命令行编译被环境限制阻断：wrapper 下载 Gradle 时提示网络权限不足；使用已缓存 Gradle 时提示无法建立 loopback 连接。需要在 Android Studio 中点击构建确认。

## 如何运行或测试这个版本

1. 在 Android Studio 中同步并构建项目。
2. 安装到 Android 手机或模拟器。
3. 桌面打开“传感器测试”入口。
4. 检测编号默认使用 `006A0001`。
5. 点击“认证”，确认状态显示认证成功。
6. 点击“获取数据”，检查六路曲线是否出现。
7. 在“六路曲线”页用双指缩放、拖动曲线、底部滑块平移，并勾选/取消 S1-S6。
8. 切换到“损伤列表”，点击表头测试排序，确认不同损伤等级有颜色区分。
