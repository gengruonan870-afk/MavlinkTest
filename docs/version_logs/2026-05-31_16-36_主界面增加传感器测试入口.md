# 2026-05-31 16:36 主界面增加传感器测试入口

## 本次修改目标

解决手机上“传感器测试”独立图标打开后仍进入旧主界面、难以找到传感器数据测试页的问题。改为在原 `MavlinkTest` 主界面顶部直接提供“传感器测试”按钮，点击后跳转到传感器数据测试页。

## 修改了哪些文件

- `app/src/main/java/com/example/mavlinktest/MainActivity.kt`
- `app/src/main/res/layout/activity_main.xml`
- `app/src/main/AndroidManifest.xml`
- `docs/version_logs/2026-05-31_16-36_主界面增加传感器测试入口.md`

## 每个文件主要改了什么

- `MainActivity.kt`：新增 `Intent` 跳转逻辑，点击 `btnSensorTest` 后进入 `SensorDataTestActivity`。
- `activity_main.xml`：顶部状态栏右侧新增“传感器测试”按钮。
- `AndroidManifest.xml`：取消 `SensorDataTestActivity` 的桌面启动入口，避免手机上出现两个图标造成混淆；该页面改为从主界面按钮进入。
- `docs/version_logs/2026-05-31_16-36_主界面增加传感器测试入口.md`：记录本次版本说明。

## 是否新增功能

是。新增从机器人主界面直接进入“传感器数据测试页”的入口。

## 是否修复 bug

是。修复手机上第二个“传感器测试”图标不可靠、可能仍打开旧主界面的问题。

## 是否还有遗留问题

安装新版后，手机桌面可能仍短暂保留旧的第二个图标缓存。建议先卸载旧 App，再重新运行安装。新版设计只需要打开 `MavlinkTest`，然后点击顶部“传感器测试”按钮。

## 如何运行或测试这个版本

1. 在手机上卸载旧的 `MavlinkTest` 和可能存在的 `传感器测试` 图标。
2. 从 Android Studio 重新运行安装 App。
3. 打开 `MavlinkTest`。
4. 点击顶部右侧“传感器测试”按钮。
5. 确认进入包含认证、检测编号、六路曲线和损伤列表的传感器数据测试页面。
