# 2026-05-31 17:12 显示损伤接口原始返回

## 本次修改目标

在传感器数据测试页的“损伤列表”页面增加 `/api/damageResult` 接口原始 JSON 返回显示区，用于判断损伤列表为空时到底是服务端返回空数据，还是 Android 解析字段不匹配。

## 修改了哪些文件

- `app/src/main/java/com/example/mavlinktest/SensorDataTestActivity.kt`
- `app/src/main/res/layout/activity_sensor_data_test.xml`
- `docs/version_logs/2026-05-31_17-12_显示损伤接口原始返回.md`

## 每个文件主要改了什么

- `SensorDataTestActivity.kt`：新增 `tvDamageRawJson` 绑定；获取 `/api/damageResult` 成功后，将原始 JSON 格式化并显示到页面；新增 JSON 格式化辅助方法。
- `activity_sensor_data_test.xml`：在损伤列表页下方新增“接口返回 /api/damageResult”区域，可横向和纵向滚动查看原始返回文本。
- `docs/version_logs/2026-05-31_17-12_显示损伤接口原始返回.md`：记录本次版本说明。

## 是否新增功能

是。新增损伤接口原始 JSON 查看功能。

## 是否修复 bug

否。本次主要用于调试和确认接口返回内容。

## 是否还有遗留问题

如果原始 JSON 中 `data` 为空，则需要向师兄确认当前 `serial` 是否确实有损伤结果；如果 JSON 有数据但列表仍为空，则需要根据实际字段结构调整解析逻辑。

## 如何运行或测试这个版本

1. 重新运行安装 App。
2. 打开 `MavlinkTest`，点击顶部“传感器”。
3. 点击“获取数据”。
4. 切换到“损伤列表”。
5. 查看下方“接口返回 /api/damageResult”区域，确认接口返回 JSON 原文。
