# 2026-06-05 16:23 新增电压温度回读显示

## 本次修改目标

根据 `prompt.txt` 的要求，在现有 Android 地面站主界面中新增下位机电压与温度回读功能，参考 `monitor_motor.py` 的 MAVLink 命令定义解析 `31034` 与 `31033`。

## 修改了哪些文件

- `app/src/main/java/com/example/mavlinktest/MainActivity.kt`
- `app/src/main/res/layout/activity_main.xml`

## 每个文件主要改了什么

### MainActivity.kt

- 新增 `CMD_MOTOR_TELEMETRY = 31034`，用于解析电压、驱动板温度、电机温度。
- 新增 `CMD_VOLTAGE_WARNING = 31033`，用于解析电压告警等级。
- 新增电压、驱动1、电机1、驱动2、电机2的状态变量与 UI 刷新逻辑。
- 新增 3 秒无 `31034` 数据时显示 `--` 的离线保护逻辑。
- 新增温度颜色规则：
  - 驱动温度大于等于 85℃ 显示橙色，大于等于 95℃ 显示红色加粗。
  - 电机温度大于等于 65℃ 显示橙色，大于等于 75℃ 显示红色加粗。
- 新增电压告警颜色规则：
  - 等级 0 显示绿色。
  - 等级 1 显示橙色。
  - 等级 2 显示红色加粗。

### activity_main.xml

- 在顶部状态栏下方新增一行横向状态显示区域。
- 显示内容包括：
  - 电压
  - 驱动1
  - 电机1
  - 驱动2
  - 电机2
- 使用横向滚动容器，避免小屏幕文字挤压。

## 是否新增功能

是。新增了下位机电压、驱动板温度、电机温度的 MAVLink 回读解析与界面显示。

## 是否修复 bug

否。本次主要是新增功能，没有针对已知 bug 做专门修复。

## 是否还有遗留问题

- 需要连接真实下位机确认 `31034` 和 `31033` 实际回传频率与参数顺序是否完全一致。
- 当前显示区域放在主界面顶部，后续如平板界面空间充足，可以再做更大的仪表化显示。

## 如何运行或测试这个版本

1. 在 Android Studio 中选择 `app` 配置。
2. 连接遥控器或 Android 设备后运行安装。
3. 确认主界面顶部第二行显示 `电压:--V / 驱动1:--℃ / 电机1:--℃ / 驱动2:--℃ / 电机2:--℃`。
4. 接入下位机后，观察收到 `31034` 时是否更新电压与温度。
5. 停止下位机回传超过 3 秒后，确认温度显示恢复为 `--`。
6. 本地已执行 `.\gradlew.bat :app:assembleDebug`，编译通过。
